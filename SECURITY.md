# 🔐 Phantom-FS 安全模型

> **柯克霍夫原则（Kerckhoffs's Principle）**：一个真正安全的系统，即使架构、算法、源代码全部开源，只要密钥不泄露，它依然是绝对安全的。
>
> Phantom-FS 从第一行代码开始，就是基于这个最悲观的**白盒假设（White-box Assumption）**来构建的。

---

## 攻击路径推演

### 🛡️ 路径一：正面硬刚密码学（数学层）

**破解概率：绝对的 0%**

黑客拿到了 `.ptm` 文件和所有 Chunks，并且对着 GitHub 上的源码一行行看。他知道我们用了 `AES-256-GCM`，知道 IV 是怎么推导的。

**困境**：没有主密钥。暴力穷举 256 位 AES 密钥，即使调用全地球所有超级计算机和比特币矿机，算到太阳熄灭、宇宙热寂，也算不完其中的亿分之一。

**结论**：即使是国家级黑客（NSA），在没有量子计算机取得革命性突破的前提下，**正面硬解 AES-256 的可能性为物理上的绝对零。**

### 🩸 路径二：降维打击"密码薄弱点"（人性层）

**破解概率：10% ~ 90%（取决于密码强度）**

黑客转向系统里唯一的生物学漏洞——**用户脑中的密码**。`PBKDF2-SHA256` 600,000 次迭代已是军工级防御，但防不住弱密码。

**攻击方式（字典攻击）**：将最常用的 1 亿个密码逐一投入 600,000 次迭代算法中碰撞。

**结果**：
- 8 位生日密码 → 几天内必破
- 16 位随机乱码（如 `x9$Km2P!zLq4@wE`）→ 计算成本使其在物理上不可能

**结论**：密码强度足够时，此路依然是死路。

### 🔪 路径三：终端物理劫持（环境层）

**破解概率：依赖终端安全性**

黑客不碰密文，直接攻击**正在解密的用户**。

1. **内存 Dump**：播放中的明文存在于内存窗口期。若设备已被植入高级木马，可在播放瞬间 Dump 内存。
2. **键盘记录器**：底层键盘记录木马捕获密码输入，有了密码则 0% 变为 100%。
3. **供应链投毒**：钓鱼网站伪装为官方入口，骗取密码和图纸。

## 安全边界矩阵

| 攻击面 | Phantom-FS 防御 | 剩余风险 |
|--------|----------------|----------|
| AES-256-GCM 密文破解 | 数学上不可能（2^256 空间） | 0% |
| PBKDF2 600k 暴力穷举 | 计算成本极高 | 取决于密码强度 |
| 源代码完全开源 | Kerckhoffs 原则：密钥不泄露即安全 | 0% |
| .ptm 图纸泄露 | 无密码 = 废纸 | 0% |
| Chunks 泄露 | 无图纸 + 无密码 = 白噪声 | 0% |
| 内存 Dump | 播放中的明文窗口期 | 依赖 OS 安全 |
| 键盘记录器 | 无硬件密钥保护 | 依赖终端安全 |
| 钓鱼网站 | 用户识别能力 | 依赖用户警惕 |
| 物理胁迫（5 美元扳手攻击） | 无法防御 | 100% |
